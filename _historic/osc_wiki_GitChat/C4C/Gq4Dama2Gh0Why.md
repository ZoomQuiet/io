在开始课程之前得嗯哼一下:

- 俺的课程有别其它课程
- 坚决不直接给出答案, 除非迫不得已
- 坚定的认为正确的问题比正确的答案价值要高
    + 因为一个正确的问题, 代表着一个正确的自学路径, 代表一系列可以自主解决并 got 到的知识点
        * 而每个知识点, 又都依赖在一个完备的知识树上
        * 从而能帮助我们自主的, 思维清晰的解决一大票相关/同领域问题
    + 而一个正确的答案, 最多解决特定场景中一系列条件下的有问题状态修正, 
        * 而为什么会陷入这种状态, 才是更加重要的知识点
        * 可答案不可能引导我们回溯, 真正理解问题发生的环境/原因
        * 从而解决一大批同类/相似问题
- 综上, 这一课程, 只是拿 github 当话题
    + 解决 github 正确的入门姿势这一问题之上
    + 才是课程真正的目的:
        * 重新发现自己自学的能力
        * 明确正确提出问题的姿势
        * 能基于 github 的体验, 开拓其它一切相关项目托管环境中的基本技巧
    + 简单的说, github 只是个工具
        * 操练之, 带来的应该是我们对互联网服务的全新认知
        * 从而获得自在的理解任何一种在线服务的特点, 
        * 并结合自己的需求, 第一时间构造吻合自己期待的定制工具链

## 相比同类好在哪?
以上都是玩笑...

无论历史/功能/团队/... gh 其实都不算第一,
但是, 使用人数最多, 这一点就足以击破一切条件了...

毕竟, 一个技术人员, 连 gh 帐号都没有, 好意思出来嘛?

从结果上说:

- gh 变成了开源软件/工具/模块/系统/产品/思想 的一个发布中心
- 所有活跃的开发活动都在 gh 中聚集
- 这导致了异常明确的马太效应, 越发的对同类活动有吸引力

从过程上说:

- gh 是一个经典的纯粹的:
    + 由程序猿创建
    + 服务于程序猿
    + 被程序猿自发繁荣
    + 的友好场所...
- 体验上:
    + gh 比以往各种同类平台都要快捷
    + 程序小自甚至于非程序猿, 一样可以无学习用起来
    + 基础功能非常精简并坚固
    + 持续的上线实用小功能
- 文化上:
    + 从一开始就有一只形象明确的吉祥物
    + 并通过程序猿熟悉的渠道
    + 持续的进行丰富的文化活动/创造
    + 以至变成了一个专用门类的创作主题
    + [The Octodex - GitHub Octodex](https://octodex.github.com/)
    + 每年仅仅是相关周边的销售也不比 google doodle 小了

章鱼喵来自于git的作者李那厮·掏袜子同学的故乡芬兰的一个已经失传的传说. 

北欧自古产海盗. 海盗们最怕的事情就是在海上遇到漩涡,风暴什么的. 而在古时候海盗们认为是海怪作祟(比如大章鱼,参见加勒比海盗等电影). 即使是幸存,也有可能会背上海怪的诅咒. 海盗中所以流传一种说法,就是出海的时候在船上带着很多喵. 当遇上海怪的时候,就把喵扔到海怪的嘴里吃掉. 因为喵星人都是恶灵的好朋友,所以吃了喵的海怪就会进入眩晕状态从而可以逃走. 但是后来有一次海盗们遇到了喵头章鱼身的海怪,是被喵诅咒的海怪变化而成,也不再怕吃喵了,所以以后海盗们再带着喵也没有用了,从此北欧海盗迅速衰落,但这个传说没有再广泛流传. 

李那厮同学在完成git后去度假的时候,在海上就遇到了风暴九死一生,同行的人给他讲了这个传说. 所以为了平复章鱼喵的诅咒,李那厮就把章鱼喵做为github的吉祥物,通过萌化其形象来消除诅咒. 据说他从此再也没有在海上遇到风暴了.

## 挫在哪?

团队太小, 一次抢火车票事件, 就能 DDoS 挂掉..

以及, gh 真心是由纯粹的程序猿开发/运营/维护的,
从来没有什么大型推广/宣传, 
而且也养成了偷偷上线新功能的毛病, 
而且上线了也不告诉你, 
完全由熟客自行发现.
